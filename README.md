---

## 📖 什么是活书？

**一句话定义**：活书是一个封装了作者核心理论模型的、可与用户个人现实进行无限次交互的AI软件。

**核心问题**：终结"学了用不上"的知识-行动鸿沟。

**核心价值主张**：
```
活书 = 将死的书籍 → 活的AI导师
      = 外部智慧(大师理论) + 内部智慧(个人经验) → 智慧涌现
      = 终结"学了用不上"的知识鸿沟
```

**技术核心**：双引擎架构（EWAT外部智慧 + AHT内部史官）→ 智慧涌现

## 🤝 贡献指南

### 文档组织原则
1. **保留思想完整性** - 不删除任何独特思考，即使是矛盾的观点也保留
2. **去除纯重复** - 只删除逐字逐句100%相同的段落
3. **保留细微差异** - 相似但有细微差异的内容都保留
4. **清晰索引** - 用目录结构和交叉引用而非删减来组织

### 新增文档时
- 判断属于哪个目录
- 更新该目录的README
- 在根README中添加链接（如需要）

### 项目特点
- 🧠 **思想密集型**：大量哲学思考、理论推演、假设验证
- 🌀 **探索性强**：多个MVP方向并行思考
- 📝 **迭代快速**：从V1.0到V2.0，概念快速演进
- 🔄 **发散收敛并存**：既有天马行空的构想，也有务实的技术方案
