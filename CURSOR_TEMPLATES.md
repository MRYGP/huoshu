# CURSOR指令模板库

> 用于Claude快速引用的标准化指令模板
> 最后更新: 2024-12-24

---

## 使用方法

Claude在需要CURSOR执行标准任务时，使用模板引用格式：
```
T[模板编号][参数列表]
```

CURSOR收到引用后，查找对应模板并执行完整指令。

---

## 📝 文档操作模板

### T001: 创建文档
**用法**: `T001[path, title, from_docs]`

**完整指令**:
```
创建文档: {path}
标题: {title}
内容来源: {from_docs}

提取规则:
- 保留原文所有独特内容
- 只删除100%逐字逐句相同的段落（3段以上）
- 相似但有细微差异的内容都保留
- 添加来源标注

格式要求:
- 使用Markdown标准格式
- 保持逻辑清晰，层次分明
- 添加必要的目录
```

**示例**:
```
T001[
  path=00_核心理念/活书的本质.md,
  title=活书的本质,
  from_docs=[活书简单介绍.md, 活书.md]
]
```

---

### T002: 归档文档
**用法**: `T002[files_list]`

**完整指令**:
```
移动以下文件到 99_归档/:
{files_list}

操作步骤:
1. 移动文件到归档目录
2. 在每个文件开头添加废弃标注:
```
   > **状态**: 已废弃
   > **替代文档**: [新文档路径]
   > **归档时间**: YYYY-MM-DD
   > **归档原因**: [简要说明]
```
3. 更新99_归档/README.md，添加归档记录
```

**示例**:
```
T002[files=[活书.md, 活书简单介绍.md, 活书MVP.md]]
```

---

### T003: 记录对话
**用法**: `T003[date, topic, key_points]`

**完整指令**:
```
任务: 生成讨论记录
对话范围: 用户会提供完整对话内容
输出路径: 04_思考日志/{date}_{topic}.md
使用模板: DISCUSSION_RECORD_TEMPLATE.md

提取重点:
{key_points}

处理要求:
- 使用标准讨论记录格式
- 保留关键数据、结论、决策
- 突出风险和行动项
- 添加文档交叉引用
- 生成简明的目录
```

**示例**:
```
T003[
  date=2024-12-24,
  topic=Token优化系统设计,
  key_points=[简洁模式, 模板系统, 状态管理, 对话归档革命]
]
```

---

### T004: 更新README
**用法**: `T004[dir_path, update_type, content]`

**完整指令**:
```
更新目录README: {dir_path}/README.md
更新类型: {update_type}
  - add_doc: 添加新文档到列表
  - update_status: 更新文档状态
  - add_section: 添加新章节

内容: {content}

要求:
- 保持现有格式风格
- 更新"最后更新"时间戳
- 确保链接有效
```

**示例**:
```
T004[
  dir_path=02_MVP探索/方向B_谋攻体验器,
  update_type=add_doc,
  content="产品方案v2.0.md - 融入健身房模式的新版本"
]
```

## 🔄 状态管理模板

### T101: 更新项目状态
**用法**: `T101[field, new_value, reason]`

**完整指令**:
```
更新 PROJECT_STATE.md

字段: {field}
新值: {new_value}
原因: {reason}

操作步骤:
1. 找到对应字段
2. 更新值
3. 在"最近重要变更"中添加记录:
   - [YYYY-MM-DD] {field}: {old_value} → {new_value}
   - 原因: {reason}
4. 更新"最后更新"时间戳
```

**示例**:
```
T101[
  field=当前最大风险,
  new_value=低频使用导致续费率低,
  reason=商业模式验证讨论中识别
]
```

---

### T102: 记录重大决策
**用法**: `T102[decision_title, decision_content]`

**完整指令**:
```
在 PROJECT_STATE.md 的"最近重要决策"部分添加:

- [YYYY-MM-DD] {decision_title}
  简要说明: {decision_content}
  详见: [对应的ADR或讨论记录]

同时创建对应的ADR文件:
05_决策记录/ADR-XXX_{decision_title}.md
```

**示例**:
```
T102[
  decision_title=采用健身房模式解决低频问题,
  decision_content=通过每日思维训练提高用户使用频次，从月1-2次提升到周3-4次
]
```

## 🏗️ 结构操作模板

### T201: 创建目录结构
**用法**: `T201[base_path, structure]`

**完整指令**:
```
在 {base_path} 下创建目录结构:
{structure}

为每个目录创建README.md说明文件
内容包括:
- 目录用途
- 文档列表（初始为待创建）
- 使用说明
```

**示例**:
```
T201[
  base_path=02_MVP探索/方向D_新方向,
  structure=[产品方案/, 技术方案/, 思考笔记/]
]
```

## 📋 批量操作模板

### T301: 批量重命名
**用法**: `T301[pattern, replacement, file_list]`

**完整指令**:
```
批量重命名文件:
匹配模式: {pattern}
替换为: {replacement}
文件列表: {file_list}

操作后:
- 更新所有引用这些文件的链接
- 记录重命名日志
```

## 🔧 自定义模板

用户可以添加自定义模板，格式：
```markdown
### T[4位数字]: [模板名称]
**用法**: `T[编号][参数]`
**完整指令**: ...
**示例**: ...
```

建议自定义模板编号从T9001开始，避免与系统模板冲突。
